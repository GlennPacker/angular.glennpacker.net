The Basic Angular Component

The angular component is the basic building block of the domain specific DOM. Each component is capable of taking in data and displaying a business specific item.

@Component

@Component is where all items for the component are registered.

selector - This is the name that will be used in the html by other components to display this component. From the example code this would be <app-component-name></app-component-name>

templateUrl - The path to the components html markup file.

template - Html markup inline markup. For multi line use back ticks '`'

styleUrls - An array of css style files used by the component

styles - Inline version for all the styles used by the component

            
    @Component({{'{'}}'
        selector: 'app-coponent-name',
        templateUrl: './componentName.component.html',
        styleUrls: ['./componentName.component.scss']
`    {{'}'}})

Components with inputs

The basic component with no inputs has limited use as it has no data (exception being when using Ngrx). Therefore we need to inject or bind data one of the easiest ways to do this is using component input parameters.

The calling or parent component we use a snippet like the below, where we see a component calling 'my-component'.

The data inputs have square brackets to inject data into the child component.

            
                <app-my-component
                    [foo]="bar"
                >
                </app-my-component>
            
        
            
        import {{'{'}} Component, Input {{'}'}} from '@angular/core';

        @Component({{'{'}}
            selector: 'app-my-component',
            templateUrl: './my-component.component.html'
        {{'}'}})
        export class MyComponentComponent {{'{'}}
            @Input() foo: string;
        {{'}'}}
Within the recieving component use the input decorator which is referenced from angular/core.

Event Emitter and Output decorator

It common for a component to do somthing like a click or updating data. Which the parent or calling component needs to know about. This is where EventEmitter and Output decorator comes in.

The process starts with an injected function from the parent to the child. The '()' indicating the outgoing binding and '$event' the passed data from child to parent. 
            
                <app-my-component
                    ()="parentFunctionCall($event)"
                >
                </app-my-component>
            
        
            
        import {{ '{' }} Component, Output, EventEmitter {{ '}' }} from '@angular/core'; 
        
        @Component({{ '{' }} 
            selector: 'app-my-component',
            templateUrl: './sidebar.component.html', 
        {{'}'}})  
        
        export class MyComponentComponent{{ '{' }}
            @Output() clickEvent = new EventEmitter<boolean>(); 
        
            onClick() {{'{'}}
                this.clickEvent.emit(true);
            {{ '}' }}
        {{ '}' }}

The child recieves the injected function in a similar way to the input. This time using the '@Output()' decorator. This needs to be a callable method therefore the EventEmitter is used. The type is the data type to be sent.

The data/ event is then fired by calling the emit method.
